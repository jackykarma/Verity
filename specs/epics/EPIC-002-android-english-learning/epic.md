# EPIC 规格说明：Android 端英语学习 APP

**EPIC ID**：EPIC-002
**EPIC Version**：v0.1.0
**创建时间**：2026-01-25
**状态**：草稿
**输入**：`"编写一个android端的英语学习的APP，核心功能包括单词记忆系统、听力训练、阅读训练、口语训练、设置（可以切换支持英式发音、美式发音）"`

## 背景、目标与价值 *（必填）*

- **背景**：用户希望在 Android 设备上通过系统化的听、说、读、记训练提升英语能力，并可按偏好选择英式或美式发音。
- **目标**：交付一款可独立使用的 Android 英语学习应用，覆盖单词记忆、听力、阅读、口语四大训练模块，以及可影响全局发音的设置能力。
- **价值**：为学习者提供一站式、可定制的英语练习环境，提升留存与学习效率。
- **产品竞争力**：四大核心训练模块联动、发音偏好可配置，满足不同口音需求。
- **技术竞争力**：端侧为主、可离线使用核心能力，发音与语音能力可复用于多模块。
- **成功标准（EPIC Level）**：用户可在 30 天内持续使用至少 2 种训练模块；发音偏好切换后，相关模块的语音输出在 1 次操作内生效。

## 范围 *（必填）*

- **In Scope**：
  - 单词记忆系统（识记、复习、进度管理）
  - 听力训练（音频播放、题目/练习、结果反馈）
  - 阅读训练（文本阅读、理解练习、反馈）
  - 口语训练（跟读/复述、录音、反馈或评分）
  - 设置：发音偏好（英式/美式）的切换与持久化，供上述模块使用
- **Out of Scope**：
  - 多语种（仅英语）、社交/社区、付费与内购逻辑、教师端或后台运营、Web 端或 iOS 端
- **上项范围**：
  - 地区：国内优先，可扩展海外
  - 销售渠道：应用商店及自有渠道
  - 设备类型：手机、平板
  - 机型定位：中端及以上
  - OS 版本：Android 8.0+ / API Level 26+
  - 保密等级：内部
- **上项节奏**：首版覆盖四大训练模块与设置，后续按数据与反馈迭代。

## 约束与假设 *（必填）*

- **约束**：依赖 Android 系统 TTS 或第三方语音引擎实现英/美发音；需麦克风权限用于口语训练；存储与网络依设备与策略。
- **假设**：用户具备基础识字与操作能力；设备支持所需传感器与权限；首次发布不强制全量产品埋点，可后续迭代；发音偏好缺省为美式。

## 依赖关系 *（必填）*

- **上游依赖**：系统 TTS 或第三方语音/识别服务、设备麦克风与扬声器、本地存储。
- **下游影响**：单词、听力、口语等使用语音与发音偏好的模块。
- **外部依赖故障模式**：TTS/语音服务不可用或超时时，相关功能降级为仅文本或提示用户检查权限/网络。

### Feature 边界说明（每个 Feature 必填）

#### Feature: 设置（含英式/美式发音切换）

- **类型**：Product
- **范围（In/Out）**：
  - In：发音偏好（英式/美式）的 UI 切换、持久化与提供查询；基础设置入口与导航。
  - Out：主题、字体、通知、账号、多语种等其它设置；TTS 引擎的实现与选型归属能力层或各业务 Feature。
- **依赖**：无
- **验收意图**：用户可在一处切换英式/美式并在退出后仍生效；使用发音的模块能读到最新偏好。
- **拆分动机**：发音偏好为跨模块能力，先行交付可减少后续返工。
- **建议下一步**：运行 `/speckit.feature`，输入：`设置：提供发音偏好（英式/美式）的切换、持久化与查询，以及基础设置入口。`

#### Feature: 单词记忆系统

- **类型**：Product
- **范围（In/Out）**：
  - In：词库或词单、识记流程、复习与遗忘曲线（或简易复习）、学习进度与统计。
  - Out：词库的云端同步、UGC 词单、跨设备；具体算法实现放在 Feature spec。
- **依赖**：设置（读取发音偏好，用于单词发音播放）
- **验收意图**：用户可完成选词、识记、复习闭环；发音播放遵循全局发音偏好。
- **拆分动机**：背词为高频刚需，可独立交付与验证。
- **建议下一步**：运行 `/speckit.feature`，输入：`单词记忆系统：词库与词单、识记、复习与进度管理，单词发音需遵循设置中的英式/美式偏好。`

#### Feature: 听力训练

- **类型**：Product
- **范围（In/Out）**：
  - In：听力材料（音频+题目或练习）、播放控制、答题或跟听、结果与反馈。
  - Out：听力材料的 UGC、实时流式音频、多人口语识别。
- **依赖**：设置（发音偏好，若材料为合成或统一口音）
- **验收意图**：用户可听材料并完成配套练习；材料发音与全局偏好一致时体验符合预期。
- **拆分动机**：输入型训练，与阅读、口语形成闭环。
- **建议下一步**：运行 `/speckit.feature`，输入：`听力训练：听力材料播放、题目或练习、结果反馈；音频发音需支持并遵循设置中的英式/美式偏好。`

#### Feature: 阅读训练

- **类型**：Product
- **范围（In/Out）**：
  - In：阅读文章、理解题或练习、释义/生词、结果反馈。
  - Out：阅读材料的 UGC、自适应难度、多模态（纯图/视频）。
- **依赖**：无（若提供文章朗读，则依赖设置中的发音偏好）
- **验收意图**：用户可完成阅读与配套理解练习；若有朗读，则遵循发音偏好。
- **拆分动机**：与听力、单词形成输入与词汇应用闭环。
- **建议下一步**：运行 `/speckit.feature`，输入：`阅读训练：阅读材料、理解练习、生词与反馈；若支持朗读则遵循设置中的英式/美式发音偏好。`

#### Feature: 口语训练

- **类型**：Product
- **范围（In/Out）**：
  - In：跟读/复述、录音、回放；可选：简单评分或完成反馈。
  - Out：全句纠音、多轮对话、复杂口语测评模型。
- **依赖**：设置（发音偏好，用于示范音播放）
- **验收意图**：用户可跟读或复述并听到自己的录音；示范音遵循发音偏好。
- **拆分动机**：输出型训练，与听力、单词形成闭环。
- **建议下一步**：运行 `/speckit.feature`，输入：`口语训练：跟读/复述、录音与回放，可选简单评分；示范发音需遵循设置中的英式/美式偏好。`

> - 可以作为“Capability Feature”出现在 Feature 列表中，但其设计建议优先集中在本节。

> 强制做法（必须遵守）：

> - **算法能力**：**每一个算法能力/模型**必须单独创建一个 **Capability Feature**（例如 `算法能力：XXX模型端侧推理与SDK`）。该 Feature 必须覆盖：模型产物提供、推理部署（端侧/服务端）、工程化封装（SDK/接口）、评估与回退策略。

## 整体 FR / NFR（EPIC Level）*（必填）*

> 说明：
> - 这里记录 EPIC 级别的“整体需求/约束”，用于统一方向与验收口径（例如全链路性能、功耗上限、隐私合规）。
> - Feature 的 FR/NFR 仍在各自 `spec.md` 中详细化，但必须与本节保持一致，不得冲突。

### EPIC-FR（跨 Feature 的功能目标）

- **EPIC-FR-001**：用户可在单机完成单词记忆、听力、阅读、口语四种训练的核心流程，无需强制登录或联网。
- **EPIC-FR-002**：用户可在设置中切换英式/美式发音，且单词、听力、口语中与发音相关的输出在下次使用或一次刷新内生效。
- **EPIC-FR-003**：各训练模块具备可理解的进度或结果反馈，使用户能判断完成度与改进方向。

### EPIC-NFR（跨 Feature 的非功能约束）

- **EPIC-NFR-PERF-001**：应用冷启到首屏可交互不超过 3 秒（中端机、常温）。
- **EPIC-NFR-PERF-002**：音频开始播放的端到端延迟不超过 2 秒（本地或已缓存资源）。
- **EPIC-NFR-MEM-001**：常规学习会话下内存占用不超过 300MB（P50，中端机）。
- **EPIC-NFR-SEC-001**：敏感权限（麦克风、存储）须按最小范围申请，并在使用前有明确场景说明；用户语音仅本地处理，不未经同意上传。
- **EPIC-NFR-OBS-001**：首版可仅做关键流程的轻量可观测（如崩溃、启动）；产品埋点可按迭代补充，不做强制 Capability Feature。
