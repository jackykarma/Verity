## 复杂度调整因子

### 经验水平调整

| 经验水平 | 调整系数 | 说明 |
|----------|----------|------|
| 初级（< 1 年） | 1.5-2.0x | 需要大量学习和试错 |
| 中级（1-3 年） | 1.0-1.3x | 基本熟悉，偶尔需要查阅 |
| 高级（> 3 年） | 0.8-1.0x | 经验丰富，效率高 |

### 技术熟悉度调整

| 熟悉度 | 调整系数 | 说明 |
|--------|----------|------|
| 完全陌生 | 2.0-3.0x | 需要从零学习 |
| 听说过 | 1.5-2.0x | 需要深入学习 |
| 用过类似技术 | 1.2-1.5x | 需要适应新框架 |
| 非常熟悉 | 1.0x | 直接开发 |

### 代码质量要求调整

| 质量要求 | 调整系数 | 包含内容 |
|----------|----------|----------|
| MVP 原型 | 0.6-0.8x | 快速验证，质量要求低 |
| 标准质量 | 1.0x | 常规代码规范 |
| 高质量 | 1.3-1.5x | 完善测试、文档、代码审查 |
| 生产级 | 1.5-2.0x | 极高可靠性、性能优化 |

### 团队协作因素

| 因素 | 调整系数 | 说明 |
|------|----------|------|
| 独立开发 | 1.0x | 基准 |
| 需要频繁沟通 | 1.1-1.2x | 接口对接、需求确认 |
| 依赖其他模块 | 1.2-1.5x | 等待依赖完成 |
| 多人协作同一模块 | 1.3-1.8x | 代码冲突、协调成本 |

## 功能规模参考

### 小型功能（1-3 天）
- 单一屏幕，简单交互
- 基础 CRUD 操作
- 标准组件使用
- 少量业务逻辑
- 示例：设置页面、关于页面、简单列表

### 中型功能（4-10 天）
- 多屏幕流程
- 中等复杂度算法
- 本地数据存储
- 常规业务逻辑
- 第三方库集成
- 示例：登录注册流程、搜索功能、统计报表

### 大型功能（11-30 天）
- 完整的功能模块
- 复杂业务流程
- 高级算法实现
- 复杂第三方集成
- 性能优化需求
- 示例：社交模块、支付系统、实时通讯

### 超大型功能（30+ 天）
- 多个子功能组合
- 架构级别设计
- 跨模块协调
- 高性能高可靠性
- 示例：完整的电商系统、复杂的游戏逻辑
