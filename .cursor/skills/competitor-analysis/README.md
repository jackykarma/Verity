# 竞品分析 Skill

## 概述

竞品分析 skill 是一个系统化的竞品研究工具，帮助团队从**产品功能**和**技术实现**两个维度深入分析竞品，为产品决策和技术选型提供有力支撑。

## 核心价值

### 🎯 双维度分析

- **产品功能维度**：用户体验、功能设计、交互逻辑、差异化特点
- **技术实现维度**：架构设计、技术选型、实现方案、性能优化

### 🔍 系统化方法论

- 结构化的分析流程
- 标准化的输出模板
- 可量化的评估标准
- 可操作的借鉴建议

### 📊 可视化呈现

- Mermaid 架构图和流程图
- 多维度对比矩阵
- 优先级分级
- 实施路线图

## 适用场景

### 产品规划阶段
- 新功能立项前的市场调研
- 了解行业标杆产品
- 寻找差异化方向

### 技术选型阶段
- 评估不同技术方案
- 学习成熟技术架构
- 降低技术风险

### 需求评审阶段
- 验证功能合理性
- 评估实现难度
- 提供参考案例

### 产品迭代阶段
- 追踪竞品更新
- 优化现有功能
- 寻找创新方向


## 输出内容

### 完整模式输出

一份完整的竞品分析报告包含：

1. **执行摘要**（5分钟读完核心结论）
   - 核心发现
   - 快速结论
   - 关键借鉴点

2. **产品功能分析**（深入了解产品层面）
   - 功能对比矩阵
   - 核心功能详细对比
   - 用户体验评分

3. **技术实现分析**（深入了解技术层面）
   - 整体技术架构
   - 核心技术对比
   - 关键实现方案
   - 数据流程分析
   - 性能与兼容性

4. **借鉴价值分析**（可落地的建议）
   - 高/中/低价值借鉴点
   - 不建议借鉴的方案
   - 具体实施建议

5. **差异化策略**（超越竞品的方向）
   - 竞品共性分析
   - 差异化方向建议
   - 技术创新建议

6. **风险与挑战**（提前识别问题）
   - 技术风险
   - 产品风险
   - 资源挑战

7. **实施路线图**（从分析到执行）
   - 短期目标
   - 中期目标
   - 长期目标

### 快速模式输出

适合时间紧迫的场景：

- 核心发现（3-5条）
- 功能对比表格
- 技术方案对比表格
- 优先级建议（P0/P1/P2）

## 分析维度

### 产品功能维度

| 分析点 | 内容 |
|-------|------|
| 功能完整性 | 功能清单、功能覆盖、版本对比 |
| 用户体验 | 易用性、交互设计、性能体验、视觉设计 |
| 差异化特点 | 核心亮点、创新功能、竞争优势 |
| 用户价值 | 解决的问题、使用场景、目标用户 |

### 技术实现维度

| 分析点 | 内容 |
|-------|------|
| 技术架构 | 整体架构、技术分层、数据流向 |
| 核心组件 | 客户端架构、数据存储、网络层、AI引擎 |
| 实现方案 | 算法选择、技术选型、性能优化 |
| 技术选型理由 | 为什么选这个方案、适用场景、限制条件 |

## 最佳实践

### ✅ DO

- **明确分析目的**：说明为什么要分析，用于什么决策
- **指定分析重点**：功能/技术/全面
- **提供上下文**：我们的场景、约束条件
- **使用 WebSearch**：获取最新信息
- **标注推测内容**：信息不确定时明确说明

### ❌ DON'T

- **避免主观臆断**：用数据和事实说话
- **避免片面分析**：平衡功能、技术、成本
- **避免盲目借鉴**：考虑自身场景和资源
- **避免过时信息**：使用最新资料
- **避免忽视风险**：客观评估挑战

## 常见问题

### Q: 分析几个竞品比较合适？

**A**: 建议 2-3 个进行深度对比。太少缺乏对比性，太多难以深入。

### Q: 找不到竞品技术资料怎么办？

**A**: 
1. 使用 WebSearch 搜索官方博客、技术分享
2. 从产品功能反推技术实现
3. 参考行业通用方案
4. 明确标注"推测"

### Q: 如何确保分析的客观性？

**A**:
- 使用多个信息源交叉验证
- 引用官方资料
- 提供数据支撑
- 实际测试产品
- 在报告中说明分析依据

### Q: 技术方案很复杂，如何简化？

**A**:
- 抓住核心技术点
- 使用架构图可视化
- 提供简化版说明
- 举例说明关键实现
