# SDK 文档生成器 Skill

这是一个专门用于生成 SDK 接口文档的 Cursor Agent Skill,支持多种编程语言和文档格式。

## 功能特性

- 🔍 **自动扫描 API** - 分析代码提取公开的类、方法、接口
- 📝 **多格式支持** - 生成 Markdown、KDoc/JavaDoc、OpenAPI 规范
- 💡 **示例生成** - 为关键 API 自动生成可运行的使用示例
- 📚 **完整文档包** - 生成 README、API 参考、使用指南等
- 🌍 **多语言支持** - Kotlin、Java、Python、JavaScript/TypeScript
- 🔄 **增量更新** - 智能更新现有文档

## 快速使用

当你需要生成 SDK 文档时,只需要告诉 Cursor AI:

```
"请为我的 SDK 生成 API 文档"
"生成完整的接口文档,包括使用示例"
"为 MainActivity 类生成 API 参考"
"生成 OpenAPI 规范"
```

Skill 会自动激活并引导你完成整个文档生成过程。

## 使用场景

### 1. 新建 SDK 项目

为全新的 SDK 项目生成完整的文档结构:

```
"为我的新 SDK 生成完整文档,包括 README、API 参考和使用指南"
```

### 2. 更新现有文档

当 API 发生变更时更新文档:

```
"我添加了新的方法,请更新 API 文档"
```

### 3. 生成特定类的文档

为单个类或模块生成详细文档:

```
"为 WordManager 类生成详细的 API 文档,包含使用示例"
```

### 4. 生成 REST API 规范

为 REST API 生成 OpenAPI/Swagger 规范:

```
"生成 OpenAPI 3.0 规范,包含所有端点"
```

### 5. 准备开源发布

生成开源项目所需的所有文档:

```
"准备开源发布,生成 README、贡献指南、API 文档和示例"
```

可运行的示例代码:

- 常见问题解决方案

## 最佳实践

1. **保持代码注释最新** - 文档质量依赖于代码中的注释
2. **使用描述性命名** - 清晰的类名和方法名有助于生成更好的文档
3. **定期更新文档** - 在 API 变更时及时更新文档
4. **添加使用示例** - 为复杂的 API 提供详细示例
5. **收集用户反馈** - 根据用户反馈改进文档

## 常见问题

### Q: 如何让 Skill 自动激活?

A: 当你提到 "SDK 文档"、"API 参考"、"接口说明" 等关键词时,Skill 会自动激活。

### Q: 可以生成中文文档吗?

A: 可以。在请求时指定语言即可,例如 "生成中文 API 文档"。

### Q: 生成的文档在哪里?

A: 默认在项目的 `docs/` 目录。你可以在生成时指定其他位置。

### Q: 如何处理已有的文档?

A: Skill 会智能识别现有文档并进行增量更新,而不是完全覆盖。

### Q: 支持私有 API 吗?

A: 默认只文档化公开 API。如果需要,可以明确要求包含私有 API。
